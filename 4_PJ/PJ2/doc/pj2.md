# Data Structure Project 2: 地下城与勇⼠

## 1. 项目概览

需求说明：你需要实现⼀个 “地下城探险与任务规划系统“。在这个虚构的地下城世界中，地下城由多个房间（Room）组成，房间之间通过通道（Edge）连接。每个房间可能有怪物（难度不同）、宝箱（价值不同）。玩家从⼊⼝出发，在地下城中找到Boss，收集战利品提升武⼒值，战胜Boss。

系统实现⽅式不限，发挥你的想象⼒。在每个房间中，可能存在怪物（必须标注怪物武⼒值，只有玩家武⼒值⼤于怪物时才能进⾏战⽃），战胜怪物后，玩家可以得到该房间中可能存在的宝箱，并提升武⼒值（需要标注每个房间可以提升的武⼒值），打开宝箱可以获得不同的道具（道具类型包括，头盔，护甲，靴⼦，兵器，魔法⽯。不同道具具有不同价值）⽤户的初始武⼒值默认为 10。  

本次pj不再提供默认的地下城地图，请使⽤你在数据结构课程上学习到的内容，按照你的设计定义和实现地图，需要⽀持地图的⾃动⽣成。  

## 2. 实现细节

### 2.1. 基本功能

1. 地下城地图⽣成、加载与完整表示（10 分）  
    请展示完整的地下城地图，在图上不少于20个Room，这些Room之间有着连通关系。需要显示每个Room中是否存在怪物和宝箱，如果存在，标注怪物武⼒值，宝箱价值（道具类型和可以提供的武⼒值）。注意，Boss⼀定是地图上武⼒值最⾼的怪物。  
2. 战利品信息管理 （10 分）  
    为所有战利品 / 道具建⽴⼀个树结构进⾏管理，展示当前⽤户得到的所有战利品的类别和获得的具体内容。要求每次探索Room后可以⾃动更新该战利品管理系统并⽀持收藏展示。  
3. 探索路线搜索 （10 分）  
    给定⼀个⽬标房间（例如 Boss 房），系统需要在房间图上搜索⼀条最优探索路线，要求探索次数最少。  
4. 多⽅案⽣成与路线排序 (10 分)  
    探索所有可⾏路径，并按照获得战利品的价值总量对这些路线进⾏排序。  
5. 探险⽇志与记录管理 （10 分）  
    实现⼀个“探险⽇志”功能，⽤于记录玩家已经执⾏过的探索。使⽤你觉得合适的数据结构，管理在探索过程中的：起点、终点、当前完整路径、当前⻆⾊武⼒值、总奖励、记录编号或时间戳等。  

### 2.2. 拓展功能

1. 战⽃模拟（5–10 分）  
    可以按照你的设计，为每个房间增加“战⽃”⽅式，⽐如，随机数⽐⼤⼩等，有趣的战⽃⻛格。  
2. 合理的系统实现与展示⽅式 (5-10 分)  
    可以使⽤命令⾏， web， UI等多种⽅式实现该系统，争取提⾼玩家可玩性与趣味性。  
3. Boss移动 (10 分)  
    Boss可能随时在地下城中移动到某个Room，从⽽增加该Room的难度，对探索最优路径的规划需要实时规划。
4. 其他任何可能增加⽤户体验的优化⽅法 (5-10 分)
